# 酒店图片处理 SaaS — MVP 需求文档

**版本：** v0.1  
**日期：** 2026-03-19  
**阶段：** MVP 验证

---

## 一、产品背景

酒店经营者在日常运营中需要大量高质量的客房/外立面照片用于 OTA 平台展示（如携程、美团、Booking）。传统修图流程依赖专业修图师，成本高、周期长。本产品通过 AI 图像处理技术，让酒店经营者一键完成专业级照片精修，降低成本、提升效率。

---

## 二、目标用户

- **主要用户：** 酒店经营者（中小型酒店、民宿、连锁酒店运营人员）
- **使用场景：** 更新 OTA 平台房间照片、营销物料制作、新房型上线

---

## 三、MVP 范围

### 3.1 MVP 目标
用最小成本验证核心价值：**AI 能否显著提升酒店照片质量，用户愿意使用？**

### 3.2 MVP 功能（2个）

#### 功能 1：客房一键精修
- **描述：** 对客房照片进行全图 AI 精修
- **处理内容：**
  - 室内智能打光
  - 被子/床品光滑处理（去除褶皱）
  - 去除杂物（遥控器、水杯等）
  - 统一色温（暖白平衡）
  - 降低窗外过曝
- **适用场景：** 标准客房、套房的主图精修

#### 功能 2：一键阳光
- **描述：** 为室内照片增加自然阳光效果
- **处理内容：**
  - 增加阳光光束/光晕
  - 营造温暖光影氛围
  - 提升整体亮度层次感
- **适用场景：** 白天拍摄但光线不足的客房照片

### 3.3 MVP 不包含（后续迭代）
- 修床光滑无褶皱（独立功能）
- 室内氛围蓝调夜景
- 室内氛围日落黄昏
- 外立面精修（夜晚）
- 用户注册/登录
- 付费/订阅系统
- 批量处理

---

## 四、用户流程

```
进入网站
  ↓
选择功能（客房一键精修 / 一键阳光）
  ↓
上传照片（支持 JPG/PNG，最大 10MB）
  ↓
点击「开始处理」
  ↓
等待 AI 处理（进度提示，预计 30-60 秒）
  ↓
查看对比效果（原图 / 处理后左右对比）
  ↓
下载处理后的图片
```

---

## 五、页面结构

### 5.1 主页（单页应用）
- **Header：** 产品名称 + 简短 slogan
- **功能选择区：** 2个功能卡片，点击选中
- **上传区：** 拖拽或点击上传，支持预览
- **处理按钮：** 选择功能 + 上传图片后激活
- **结果展示区：** 原图/结果图左右对比滑块
- **下载按钮：** 下载处理后图片

### 5.2 无需其他页面（MVP 阶段）

## 六、技术方案

| 项目 | 选型 |
|------|------|
| 前端框架 | Next.js 14 (App Router) |
| UI 样式 | Tailwind CSS |

| 后端 | Next.js API Routes |
| 文件存储 | 本地临时目录（MVP） |
| 部署 | 本地运行（MVP） |

### 6.1 IC-Light 模型参数映射

| 功能 | Prompt 方向 |
|------|------------|
| 客房一键精修 | `professional hotel room photography, smooth bedding, clean interior, warm lighting, no clutter, balanced exposure` |
| 一键阳光 | `bright sunlight streaming through window, warm golden light, natural sunshine atmosphere, hotel room` |

---

## 七、非功能需求

- **处理时长：** 单张图片 ≤ 90 秒（Replicate 冷启动）
- **图片格式：** 输入支持 JPG/PNG；输出 JPG
- **图片大小：** 输入最大 10MB
- **分辨率：** 保持原图分辨率输出
- **并发：** MVP 阶段不做限制（单用户本地测试）
